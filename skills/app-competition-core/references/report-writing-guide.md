# Report Writing Guide

## Goal

默认输出“混合型”竞品报告：
- 前半段给高层看：摘要、关键变化、风险、动作建议
- 后半段给研究留档：模块化事实、证据、限制说明

目标不是堆数据，而是把“本期发生了什么、为什么重要、该怎么应对”讲清楚。

## Default Structure

推荐顺序：
1. `Executive Summary`
2. `This Month's Key Moves`
3. `<Market> Market Snapshot`
4. `Release Watch`
5. `Creative / Screenshot Changes`
6. `Keyword Rank Alerts`
7. `Reviews, Pricing And Portfolio Signals`
8. `Implications, Risks, Next Actions`
9. `Evidence Appendix`
10. `Limitations`

## Core Writing Pattern

对每个重点模块，优先按 4 句式写：
1. `发生了什么`
2. `证据`
3. `为什么重要`
4. `我们该做什么`

如果模块很轻，可以压成 2-3 句，但顺序不要乱。

## Fact / Inference / Limitation

- `fact`：直接来自 MCP 返回、历史快照或截图证据。
- `inference`：基于多个事实做出的判断，要把依据点出来。
- `limitation`：缺历史、缺截图、缺版本历史、字段不完整等。

不要把 `inference` 写成 `fact`，也不要把 `limitation` 藏起来。

## Section Guidance

### Executive Summary

- 只写最重要的 3-5 个结论。
- 要能让不看后文的人也知道本期该关注什么。

### Release Watch

- 先写最近 30 天谁发版了。
- 再写 `What's New` 在说什么。
- 最后写这是能力扩张、质量修复、定价包装，还是营销动作。

### Creative / Screenshot Changes

- 优先盯前三张图。
- 不是只写“换了图”，而是写“表达重点从什么变成了什么”。
- 如果要谈市场变化，先写 `market_change_hypothesis`，再说明依据来自哪些交叉信号。

### Keyword Rank Alerts

- 不只写流量分，要写真实 rank 快照。
- 固定输出：`Top Gainers`、`Top Losers`、`New Entrants`、`Defensive Keywords`、`Opportunity Keywords`。

## Tone Rules

- 用研究口吻，不用营销腔。
- 结论要明确，但不要装作比证据更确定。
- 不要把报告写成 API 字段转抄。

## Common Mistakes

- 开头写太长的背景，真正结论埋在后面
- 只写“有变化”，不写变化阈值和影响
- 只写单个模块，不串起 `change_chain`
- 没有历史时硬写升降趋势

## Completeness Check

在交付月报前，快速确认：
- 有 `comparison_window`
- 有 `What's New`
- 有前三张截图语义摘要
- 有关键词 rank alerts
- 有至少一个 `market_change_hypothesis` 或明确写出“本期没有足够证据形成假设”
- 有 `limitations`
