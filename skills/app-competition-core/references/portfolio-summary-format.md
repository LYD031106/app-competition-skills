# Portfolio Summary Format

## Goal

Portfolio 摘要是“多自有产品的竞品字段核查入口”。默认只交付 1 份飞书摘要文档；详细子文档只有用户明确要求时才创建。

摘要必须让读者快速看清：
- 每个自有产品的 target keyword 当前状态。
- 每个自有产品当前标题 / 名称、产品描述、更新日志是否变化。
- 每个产品绑定的核心竞品 / leader 在标题、描述、竞品 iPhone 前三图、更新日志上是否变化。
- 竞品 icon 和竞品宣传图证据。

除非用户明确要求建议或分析，禁止输出建议、下一步动作、影响评估、风险判断、优先级排序、机会判断。

## Required Structure

飞书摘要固定结构：

1. 内容型标题
2. `产品关键词记录`
3. 对每个自有产品重复以下结构：
   - `# [自有产品] 关键词与竞品字段核查`
   - `## 目标产品关键词记录`
   - `## 目标产品字段核查`
   - `## [自有产品] 竞品变化核查`
   - `## [自有产品] 竞品宣传图核查`

不要加入额外的封面信息表、事实综述段、产品小摘要表、全局竞品索引、归档证据表、二级文档入口、局限章节或规则清单。竞品核查必须留在各自有产品段内。

## Section Rules

### 产品关键词记录

全局窄表，列固定为：
- `自有产品`
- `target keyword`
- `目标产品状态`
- `leader / 当前首位对象`
- `记录来源`
- `备注`

每个产品默认 1-3 行。没有真值 rank 时，`记录来源` 写 fallback 来源，不能写成真实排名。

### 目标产品关键词记录

每个产品段内再写一次该产品自己的关键词表，列固定为：
- `target keyword`
- `目标产品状态`
- `leader / 当前首位对象`
- `记录来源`
- `备注`

### 目标产品字段核查

只写三项：
- 标题 / 名称
- 产品描述
- 更新日志

产品描述必须与上一版比较：
- 无变化写 `无变化`。
- 有变化才写 `之前的产品描述：...`、`本次产品描述：...`、`改动点：...`。

更新日志直接粘贴抓取到的 releaseNotes 原文；没有则写 `没有`。

默认不要展示自有产品自己的 iPhone top1-3 宣传图。

### 竞品变化核查

每个产品段内列 1-3 个核心竞品或关键词 leader。表格列固定为：
- `icon`
- `竞品`
- `角色`
- `标题 / 名称`
- `竞品产品描述`
- `竞品 iPhone 前三图`
- `更新日志`

规则：
- `icon` 必须使用抓取到的产品 icon；没有 icon 时留空或写 `无 icon`。
- 竞品名称必须可点击，使用 canonical App Store URL。
- 竞品产品描述无变化写 `无变化`；有变化写 `之前的产品描述：...`、`本次产品描述：...`、`改动点：...`。
- 竞品 iPhone 前三图无图写 `无宣传图`。
- 更新日志直接放 releaseNotes 原文；没有写 `没有`。

### 竞品宣传图核查

只展示竞品图，不展示自有产品图。

每个竞品单独一个小节：

```markdown
### [Competitor] 宣传图

- 前三张 iPhone 宣传图变化：[无变化 / 有变化：说明顺序、主题、图中文字变化 / current baseline / 无宣传图]
- 上一版竞品宣传图：[有历史图时插入上一版竞品 iPhone top1-3；没有历史基线写“无历史基线”]
- 本次竞品宣传图：[有图时插入本次竞品 iPhone top1-3；没有图只写“无宣传图”]

`[Competitor] | competitor iPhone top1-3 | [market] | collected_at=[YYYY-MM-DD] | [current baseline / previous-current]`
```

图片必须贴近对应竞品的小节；不能集中堆到文末。

## Evidence Rules

用户可见摘要只呈现内容，不解释采集管线。

可写：
- `竞品 iPhone top1-3`
- `current baseline`
- `与 YYYY-MM-DD 基线相比无变化`
- `无宣传图`

不可写：
- `官方 screenshot`
- `官方字段`
- `字段为空`
- `网页回退`
- `needs_page_fallback`
- `page_fallback`
- `lookup`
- `scraper`
- `接口为空`
- 本地 evidence 路径

这些采集状态只能写入 snapshot、manifest、evidence notes。

## Quality Gate

交付前必须通过：
- 摘要从 `产品关键词记录` 开始，不加额外开场表格或综述段。
- 远端 fetch 能看到每个自有产品的 `# [自有产品] 关键词与竞品字段核查`。
- 每个产品段都有目标产品关键词记录、目标产品字段核查、竞品变化核查、竞品宣传图核查。
- 竞品变化表有 `icon` 列和 `竞品 iPhone 前三图` 列。
- 摘要中没有自有产品自己的宣传图小节。
- 需要展示竞品图的位置有 `<image token="...">`，或写 `无宣传图`。
- bad-terms gate 通过，用户可见正文不出现采集管线词。
- manifest 不能停在 `text_only_created`。
